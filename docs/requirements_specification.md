# Stream Player 要件定義書（仕様書）

**文書バージョン**: 1.0
**作成日**: 2026-03-02
**対象バージョン**: 現行コードベース（リファクタリング後）

---

## 1. アプリケーション概要・目的

### 1.1 概要

Stream Player は、YouTube等のストリーミングコンテンツを読み込み、任意の時間位置にマーカーを配置し、マーカー間のセグメントを定義してループ再生を行うデスクトップアプリケーションである。さらにテンポ（再生速度）とピッチ（音程）の独立制御が可能で、音楽練習・語学学習・動画分析等の用途に適する。

### 1.2 目的

- ストリーミング動画の特定区間を繰り返し再生し、効率的な学習・練習を支援する
- テンポ変更時にピッチを維持し、ピッチ変更時にテンポを維持する独立制御を提供する
- マーカー・セグメント・エフェクト設定をURL単位で自動保存し、再訪問時に復元する

### 1.3 想定ユーザー

- 楽器練習者（特定フレーズの低速ループ練習）
- 語学学習者（リスニング区間の反復再生）
- 動画分析者（特定シーンの繰り返し確認）

## 2. 機能要件

### 2.1 ストリーミング再生機能

| ID | 機能 | 詳細 |
|----|------|------|
| F-PLAY-001 | URL読み込み | YouTube等のストリーミングURLを入力し、動画を読み込む。yt-dlpによりメタデータ（タイトル・長さ）を取得する |
| F-PLAY-002 | 動画再生 | mpvエンジンによる動画・音声の再生。アプリケーションウィンドウ内に映像を埋め込み表示する |
| F-PLAY-003 | 再生/一時停止 | トグル操作で再生と一時停止を切り替える |
| F-PLAY-004 | 停止 | 再生を完全に停止する |
| F-PLAY-005 | シーク（絶対位置） | タイムライン上のクリックまたは数字キー(0-9)で動画の任意の位置にジャンプする。数字キーは動画長の0%〜90%に対応する |
| F-PLAY-006 | シーク（相対位置） | 現在位置から前後に移動する。±1秒、±5秒、±10秒の3段階を提供する |
| F-PLAY-007 | フレームステップ | 一時停止中に1フレーム単位で前進・後退する |
| F-PLAY-008 | 音量調整 | 音量を0〜100の範囲で±5%単位で調整する |

### 2.2 URL履歴機能

| ID | 機能 | 詳細 |
|----|------|------|
| F-HIST-001 | 履歴保存 | 読み込み成功したURLとタイトルを `url_history.json` に保存する |
| F-HIST-002 | 履歴表示 | ドロップダウンメニューで過去のURLを一覧表示する。タイトルがある場合はタイトルを表示し、60文字を超える場合は末尾を省略する |
| F-HIST-003 | 履歴選択 | ドロップダウンから選択すると、URL入力欄にURLが自動入力される |
| F-HIST-004 | 履歴上限 | 最大50件を保持し、最新のものが先頭に表示される。同一URLが重複する場合は古い方を削除する |

### 2.3 マーカー機能

| ID | 機能 | 詳細 |
|----|------|------|
| F-MKR-001 | マーカー追加 | 現在の再生位置にマーカーを追加する。ラベルは自動でA, B, C, ..., Z, AA, AB, ...と連番付与される |
| F-MKR-002 | マーカー削除 | 指定したマーカーを削除する。該当マーカーを参照するセグメントも連動して削除される |
| F-MKR-003 | マーカー全削除 | 全マーカーを一括削除する。ラベルカウンターもリセットされる |
| F-MKR-004 | マーカー位置変更 | タイムライン上でマーカーをドラッグして位置を変更する。ドラッグ中は専用カーソル（↔）を表示する |
| F-MKR-005 | マーカーメモ | 各マーカーにテキストメモを付与できる。フォーカスアウトまたはEnterキーで保存される |
| F-MKR-006 | ラベルスワップ | 2つのマーカーのラベル（表示名）を入れ替える。内部IDは変更されない |
| F-MKR-007 | マーカーへジャンプ | マーカーパネルのシークボタンでそのマーカー位置に再生位置を移動する |
| F-MKR-008 | カラー自動割り当て | 12色のプリセットパレットから順番に色を割り当てる。パレットは循環する |
| F-MKR-009 | タイムライン表示 | マーカーはタイムライン上に色付き三角形とラベルテキストで表示される |

#### マーカーデータ構造

```
Marker:
  id: str          -- UUID (12文字hex)。不変の内部識別子
  label: str       -- 表示ラベル (A, B, ..., AA, AB, ...)
  position: float  -- 秒単位の時間位置
  color: str       -- hex色コード
  memo: str        -- ユーザーメモ（任意）
```

### 2.4 セグメント・シーケンスループ機能

| ID | 機能 | 詳細 |
|----|------|------|
| F-SEQ-001 | セグメント定義 | 開始マーカーと終了マーカーを選択してセグメント（区間）を定義する。同一マーカー同士のセグメントは作成不可 |
| F-SEQ-002 | セグメント名 | 各セグメントに任意の表示名を付与できる。未指定の場合は開始マーカーのメモから自動補完される |
| F-SEQ-003 | バッチ追加 | テキスト入力で複数セグメントを一括追加する |
| F-SEQ-004 | セグメント削除 | 個別のセグメントをリストから削除する |
| F-SEQ-005 | セグメント並び替え | 上下ボタンでセグメントの順序を変更する |
| F-SEQ-006 | ループ開始 | シーケンスの先頭セグメントから順に再生を開始する。一時停止中の場合は自動で再生を開始する |
| F-SEQ-007 | ループ停止 | ループ再生を停止する（通常再生に戻る） |
| F-SEQ-008 | セグメントジャンプ | セグメントリストの行クリックで特定セグメントの開始位置にジャンプし、そこからループを継続する |
| F-SEQ-009 | アクティブセグメント表示 | 現在再生中のセグメントをリスト上でハイライト表示し、タイムライン上で半透明の青色領域として表示する |
| F-SEQ-010 | 境界検出 | 再生位置がセグメント終了位置の0.15秒前に達した時点で次セグメントへ遷移する |

#### ループモード

| モード | 内部値 | 動作 |
|--------|--------|------|
| Loop Sequence | `loop_sequence` | 全セグメントを順番に繰り返す（末尾→先頭に戻る） |
| Loop Single | `loop_single` | 現在のセグメントのみを無限に繰り返す |
| Play Once | `play_once` | 全セグメントを1回ずつ再生し、最後まで到達したら停止する |

#### バッチ入力フォーマット（F-SEQ-003）

```
入力形式:
  AB,CD,EF          -- カンマ区切り
  AB CD EF           -- スペース区切り
  AB/CD/EF           -- スラッシュ区切り
  AB:intro,CD:verse  -- コロンでセグメント名を指定
  混在も可: AB:intro CD/EF

解析ルール:
  - 大文字小文字を区別しない（自動で大文字化）
  - 1文字ラベル: "AB" → (A, B)
  - 多文字ラベル: "AAAB" → 既知マーカーラベルに一致する分割点を探索
  - 両方のラベルが既存マーカーに存在しない場合はスキップ
```

#### セグメントデータ構造

```
Segment:
  start_marker_id: str  -- 開始マーカーのID
  end_marker_id: str     -- 終了マーカーのID
  display_name: str      -- 表示名（任意）

※セグメントの実際の時間範囲は、2つのマーカー位置のmin/maxで正規化される
（開始マーカーが終了マーカーより後ろでも正常動作する）
```

### 2.5 オーディオエフェクト機能

| ID | 機能 | 詳細 |
|----|------|------|
| F-EFX-001 | テンポ変更 | 再生速度を0.25x〜2.0xの範囲で変更する。ピッチは維持される（mpv scaletempo2による補正） |
| F-EFX-002 | テンポプリセット | 0.5x, 0.75x, 1.0x, 1.25x, 1.5x, 2.0x のプリセットボタンを提供する。現在のテンポに一致するボタンがハイライトされる |
| F-EFX-003 | テンポスライダー | 0.25x〜2.0x を35ステップのスライダーで微調整する |
| F-EFX-004 | トランスポーズ | 音程を-12〜+12半音の範囲でシフトする。テンポは維持される |
| F-EFX-005 | トランスポーズスライダー | -12〜+12を24ステップ（1半音刻み）のスライダーで調整する |
| F-EFX-006 | トランスポーズ±ボタン | ±1半音単位で音程を変更するボタンを提供する |
| F-EFX-007 | エフェクトリセット | テンポを1.0x、トランスポーズを0半音にリセットする |
| F-EFX-008 | UI自動同期 | キーボードショートカットやスライダー操作のいずれからエフェクトが変更されても、全UIコンポーネントが `effects_changed` イベント経由で自動同期する |

#### トランスポーズ技術仕様

```
実装: lavfiフィルタ (asetrate + atempo)
計算式:
  adjusted = semitones - 1  (観測された+1半音オフセットの補正)
  pitch_ratio = 2^(adjusted / 12)
  new_rate = 48000 * pitch_ratio
  tempo_comp = 1 / pitch_ratio
  af = 'lavfi="asetrate={new_rate},atempo={tempo_comp}"'

0半音の場合: フィルタをクリア（空文字列）
```

## 3. 非機能要件・動作環境

### 3.1 動作環境

| 項目 | 要件 |
|------|------|
| OS | Windows 11（主対象）。macOS/Linuxはコード上対応しているが未検証 |
| Python | 3.10以上（`str \| None` 型構文を使用） |
| ランタイム依存 | libmpv-2.dll（プロジェクトルートに同梱） |
| ネットワーク | YouTube等のストリーミングURL解決にインターネット接続が必要 |

### 3.2 依存ライブラリ

| ライブラリ | バージョン | 用途 |
|-----------|----------|------|
| customtkinter | >= 5.2.0 | モダンなGUIフレームワーク |
| python-mpv | >= 1.0.6 | mpvメディアプレーヤーのPythonバインディング |
| yt-dlp | >= 2024.1.1 | ストリーミングURL解決・メタデータ取得 |

### 3.3 パフォーマンス要件

| 項目 | 要件 |
|------|------|
| 再生遅延 | URL読み込み〜再生開始は yt-dlp 解決時間に依存（通常2〜5秒） |
| セグメント遷移 | 境界検出閾値0.15秒以内で次セグメントにシーク |
| 自動保存 | 変更後2秒のデバウンスで書き込み。UIをブロックしない |
| タイムライン更新 | mpvのプロパティ監視からのイベント駆動。手動ポーリングなし |

### 3.4 スレッドモデル

| スレッド | 用途 |
|---------|------|
| メインスレッド | tkinter GUIイベントループ |
| URL解決スレッド | yt-dlp によるメタデータ取得（デーモンスレッド） |
| mpvコールバックスレッド | プロパティ監視コールバック（mpv内部スレッド） |
| シークスレッド | セグメント遷移時のシーク操作（デーモンスレッド） |

GUIスレッドセーフティは `widget.after(0, callback)` パターンで確保する。SequenceLooperは `threading.RLock` で内部状態を保護する。

### 3.5 アーキテクチャ

```
┌──────────────────────────────────────────────────────┐
│                     run.py                            │
│                   (エントリポイント)                     │
└──────────────┬───────────────────────────────────────┘
               │
┌──────────────▼───────────────────────────────────────┐
│                    App (src/app.py)                    │
│              アプリケーションコントローラ                  │
│    ┌─────────────────────────────────────────────┐    │
│    │              EventBus (pub/sub)              │    │
│    └─────────────────────────────────────────────┘    │
│         ▲                                     ▲       │
│    ┌────┴────────────┐  ┌─────────────────────┴──┐   │

│    │  MpvPlayer       │  │  MainWindow            │   │

│    │  MarkerManager   │  │  VideoFrame            │   │
│    │  SequenceLooper  │  │  TimelineCanvas        │   │
│    │  AudioEffects    │  │  TransportBar          │   │
│    │  LoopSettingsStore│  │  MarkerPanel           │   │
│    │                  │  │  SequenceEditor        │   │
│    │                  │  │  EffectsPanel          │   │
│    └─────────────────┘  └────────────────────────┘   │
│                                                       │
│    ┌─────────────────┐  ┌────────────────────────┐   │
│    │  utils/          │  │  config.py             │   │
│    │  time_fmt        │  │  パス設定集約           │   │
│    └─────────────────┘  └────────────────────────┘   │
└──────────────────────────────────────────────────────┘
```

**設計原則**:
- core/ と gui/ はEventBusを介して疎結合
- core モジュールはGUIに依存しない
- マーカーは不変のUUID(ID)で管理し、ラベルは表示専用
- セグメントはマーカーIDで参照し、ラベル変更に影響されない

## 4. 現在の課題・未実装・検討中の機能

### 4.1 既知の課題

| ID | 種別 | 内容 | 重要度 |
|----|------|------|--------|
| ISSUE-001 | 技術的負債 | `print()` 文によるデバッグログが各モジュールに散在。`logging` モジュールへの移行が望ましい | 低 |
| ISSUE-002 | 仕様制限 | トランスポーズ実装に+1半音オフセットの経験的補正値（`adjusted = semitones - 1`）を使用している。サンプルレート48000Hz前提のハードコードあり | 中 |
| ISSUE-003 | 堅牢性 | EventBusのエラーハンドリングが `print` + `traceback` のみ。リスナーの例外が他リスナーの実行を妨げない設計だが、エラー報告機構がない | 低 |
| ISSUE-004 | UX | URL読み込みエラー時のフィードバックが `set_error(str(e))` のみ。yt-dlpの生エラーメッセージがユーザーに表示される | 中 |
| ISSUE-005 | UX | EffectsPanelの `_set_tempo_preset` と `_reset` がイベント経由の `_sync_ui` と直接UI更新の両方を行っており、二重更新の可能性がある（動作には影響しないが非効率） | 低 |
| ISSUE-006 | クロスプラットフォーム | macOS/Linux環境でのVideoFrame埋め込み（`container=True`）が未検証 | 中 |

### 4.3 テスト観点メモ（QAエンジニア向け）

以下はテスト計画策定時に特に注意すべき観点:

1. **スレッド競合**: SequenceLooperの `_on_position_changed` はmpvスレッドから呼ばれ、RLock内でシーク操作を別スレッドに委譲する。高速なセグメント遷移時の競合に注意
2. **デバウンス動作**: マーカーの連続追加・削除時に自動保存が正しくデバウンスされるか（2秒間操作がなければ1回だけ保存）
3. **URL正規化の一貫性**: 同一動画の異なるURLバリエーション（`&t=10`, `&feature=share`, `&si=xxx`付き等）が同じ設定キーに正規化されるか
4. **セグメント境界の正確性**: 0.15秒閾値での境界検出が、テンポ変更時（再生速度が変わる）にも正しく動作するか
5. **マーカー削除の連鎖**: マーカー削除→参照セグメント削除→シーケンスイベント→タイムライン更新の一連の連鎖が正しく完了するか
6. **復元時のイベント抑制**: `_restoring = True` の期間中に自動保存がトリガーされないこと
