# Stream Player Web ユーザーストーリー

**対象アーキテクチャ**: React + TypeScript (Vite) / FastAPI / Docker
**永続化**: ブラウザ localStorage
**テストフォーマット**: 各ストーリーに BDD 形式（Given / When / Then）のテストシナリオを併記

---

## Epic 1: 動画の読み込みと再生

### US-1.1: URLから動画を読み込む

**ストーリー**: ユーザーとして、YouTube等のURLを入力して動画を読み込みたい。練習したい動画をすぐに開始できるように。

**受け入れ基準**:
- [ ] URL入力欄（`<input>`）にURLを貼り付けて Enter キーまたは「Go」ボタンで読み込み開始
- [ ] 読み込み中は「Go」ボタンが「Loading...」に変わり、入力欄が `disabled` になる
- [ ] 読み込み成功時、動画タイトルがURL入力欄の下部に表示される
- [ ] 無効なURLや解決失敗時、エラーメッセージが赤字で表示される
- [ ] YouTube URL は フロントエンドで video ID を正規表現で抽出し、YouTube IFrame Player API で直接読み込む
- [ ] video ID が抽出できない場合は、バックエンド `/api/resolve` 経由で yt-dlp によりメタデータを取得する

**テストシナリオ**:

```gherkin
Scenario: 有効なYouTube URLで動画を読み込む
  Given ユーザーがStream Player Webを開いている
  When URL入力欄に "https://www.youtube.com/watch?v=dQw4w9WgXcQ" を入力する
    And 「Go」ボタンをクリックする
  Then YouTube IFrame Player に動画が読み込まれる
    And URL入力欄の下部に動画タイトルが表示される

Scenario: 読み込み中のUI状態
  Given ユーザーがStream Player Webを開いている
  When URL入力欄にURLを入力して「Go」ボタンをクリックする
  Then 「Go」ボタンのテキストが「Loading...」に変わる
    And URL入力欄が disabled になる
    And 読み込み完了後にボタンが「Go」に戻り、入力欄が有効化される

Scenario: 無効なURLを入力する
  Given ユーザーがStream Player Webを開いている
  When URL入力欄に "invalid-url" を入力して「Go」ボタンをクリックする
  Then 赤字のエラーメッセージが表示される
    And YouTube IFrame Player は変化しない

Scenario: 空のURL入力でSubmitを試みる
  Given URL入力欄が空の状態
  Then 「Go」ボタンが disabled で表示される
    And フォームを送信しても何も起きない
```

### US-1.2: URL入力履歴

**ストーリー**: ユーザーとして、過去に入力したURLを履歴から選択したい。毎回URLを探す手間を省くために。

**受け入れ基準**:
- [ ] URL入力欄にドロップダウンで過去の入力履歴が表示される
- [ ] 履歴にはURLとタイトルが表示される
- [ ] 履歴はブラウザのリロード後も保持される（localStorage の `stream_player_url_history` に永続化）
- [ ] 履歴から選択すると動画が読み込まれる

**テストシナリオ**:

```gherkin
Scenario: 動画読み込み成功時に履歴が保存される
  Given ユーザーがStream Player Webを開いている
  When 有効なURLで動画を読み込む
  Then localStorage の url_history に当該URLとタイトルが保存される

Scenario: 履歴からURLを選択する
  Given localStorage に過去の履歴が3件存在する
  When ドロップダウンから2件目の履歴を選択する
  Then URL入力欄にそのURLが自動入力される
    And 動画の読み込みが開始される

Scenario: 履歴はリロード後も保持される
  Given 動画Aを読み込み、履歴に保存された状態
  When ブラウザをリロードする
  Then ドロップダウンに動画Aの履歴が表示される

Scenario: 同一URLの重複は古い方が削除される
  Given 履歴にURL "X" が既に存在する
  When 再度URL "X" で動画を読み込む
  Then 履歴にURL "X" は1件のみ存在する
    And URL "X" は履歴の先頭に表示される
```

### US-1.3: 基本再生操作

**ストーリー**: ユーザーとして、動画の再生/一時停止、シーク、音量調整をキーボードで行いたい。マウス操作なしで素早く制御するために。

**受け入れ基準**:
- [ ] Space / K キーで再生/一時停止をトグル
- [ ] J キーで10秒戻る、L キーで10秒進む
- [ ] ArrowLeft で5秒戻る、ArrowRight で5秒進む
- [ ] Shift+ArrowLeft で1秒戻る、Shift+ArrowRight で1秒進む
- [ ] ArrowUp で音量+5、ArrowDown で音量-5
- [ ] 0〜9キーで動画の0%〜90%地点にジャンプ
- [ ] Comma / Period キーで微小単位の前後移動（一時停止中）
- [ ] `<input>`, `<textarea>`, `<select>` にフォーカスがある時、キーボードショートカットは無効

**テストシナリオ**:

```gherkin
Scenario: Spaceキーで再生/一時停止をトグルする
  Given 動画が再生中である
    And フォーム要素にフォーカスがない
  When Space キーを押す
  Then 動画が一時停止する

Scenario: Jキーで10秒戻る
  Given 動画が再生位置 30秒にある
    And フォーム要素にフォーカスがない
  When J キーを押す
  Then 再生位置が 20秒 に移動する

Scenario: 数字キーでパーセンテージジャンプ
  Given 動画の長さが 200秒 である
    And フォーム要素にフォーカスがない
  When 5 キーを押す
  Then 再生位置が 100秒（50%）に移動する

Scenario: 入力欄フォーカス中はショートカット無効
  Given URL入力欄にフォーカスがある
  When Space キーを押す
  Then 再生状態は変化しない
    And スペース文字が入力欄に入力される
```

### US-1.4: フルスクリーン

**ストーリー**: ユーザーとして、動画をフルスクリーンで表示したい。集中して視聴するために。

**受け入れ基準**:
- [ ] F キーまたは F11 キーで Fullscreen API を使ったフルスクリーン切替
- [ ] フルスクリーン時、URL欄・タイムライン・トランスポート・下部パネルが CSS `display: none` で非表示
- [ ] Escape キーでフルスクリーン解除
- [ ] 動画エリアのダブルクリックでフルスクリーン切替
- [ ] フルスクリーン解除時（`fullscreenchange` イベント）、元のレイアウトが自動復元される

**テストシナリオ**:

```gherkin
Scenario: Fキーでフルスクリーンに切り替える
  Given 動画が読み込まれている
    And フォーム要素にフォーカスがない
  When F キーを押す
  Then document.fullscreenElement が設定される
    And URL欄・下部パネルが非表示になる

Scenario: Escapeキーでフルスクリーンを解除する
  Given フルスクリーン表示中である
  When Escape キーを押す
  Then フルスクリーンが解除される
    And URL欄・下部パネルが再表示される

Scenario: 動画エリアのダブルクリックでフルスクリーン切替
  Given 動画が読み込まれている
  When 動画エリアをダブルクリックする
  Then フルスクリーン状態がトグルされる
```

## Epic 2: マーカー管理

### US-2.1: マーカーを追加する

**ストーリー**: ユーザーとして、動画の任意の時間位置にマーカーを追加したい。ループ区間の開始/終了地点を記録するために。

**受け入れ基準**:
- [ ] M キーまたはマーカーパネルの「+ Add Marker」ボタンで現在位置にマーカー追加
- [ ] マーカーにはアルファベット順のラベル（A, B, C, ...）が自動付与される
- [ ] マーカーには一意のUUID IDが内部的に割り当てられる
- [ ] マーカー追加後、タイムラインコンポーネント上にマーカー位置が表示される
- [ ] マーカーパネルのリストにマーカーが表示される（ラベル、時刻、メモ）

**テストシナリオ**:

```gherkin
Scenario: Mキーでマーカーを追加する
  Given 動画が再生位置 15.5秒 にある
    And フォーム要素にフォーカスがない
  When M キーを押す
  Then ラベル "A" のマーカーが位置 15.5秒 に追加される
    And タイムライン上にマーカーが表示される
    And マーカーパネルに "A | 0:15" が表示される

Scenario: 連続でマーカーを追加すると順番にラベルが付与される
  Given マーカーが0件の状態
  When マーカーを3回追加する
  Then ラベル "A", "B", "C" のマーカーが順に作成される

Scenario: 26個以上のマーカーを追加すると二文字ラベルになる
  Given マーカーが26個（A〜Z）存在する
  When 27個目のマーカーを追加する
  Then ラベル "AA" のマーカーが作成される
```

### US-2.2: マーカーを編集・削除する

**ストーリー**: ユーザーとして、マーカーのラベル交換やメモの編集、不要なマーカーの削除をしたい。マーカーを整理するために。

**受け入れ基準**:
- [ ] マーカーパネルで2つのマーカーのラベルを交換できる（Swap機能）
- [ ] ラベル交換してもセグメントのマーカーID参照は維持される
- [ ] マーカーにメモ（テキスト）を追加・編集できる
- [ ] メモはセグメント追加時の名前自動入力に使われる
- [ ] × ボタンでマーカーを削除できる
- [ ] 削除後、残りのマーカーのラベルは変更されない（ギャップが残る）
- [ ] マーカーをクリックするとその時間位置にシークする

**テストシナリオ**:

```gherkin
Scenario: マーカーのラベルをスワップする
  Given マーカー A（ID: "aaa"）とマーカー B（ID: "bbb"）が存在する
    And セグメント (startMarkerId: "aaa", endMarkerId: "bbb") が存在する
  When マーカー A と B のラベルをスワップする
  Then マーカー ID "aaa" のラベルが "B" になる
    And マーカー ID "bbb" のラベルが "A" になる
    And セグメントの startMarkerId / endMarkerId は変更されない

Scenario: マーカーを削除すると参照セグメントも削除される
  Given マーカー A と B が存在する
    And セグメント A→B が存在する
  When マーカー A の × ボタンをクリックする
  Then マーカー A が削除される
    And セグメント A→B も削除される
    And マーカー B のラベルは "B" のまま維持される

Scenario: マーカーのメモを編集する
  Given マーカー A が存在する
  When マーカー A のメモ欄に "イントロ" と入力する
    And メモ欄からフォーカスを外す（blur）
  Then マーカー A のメモが "イントロ" として保存される

Scenario: マーカーをクリックしてシークする
  Given マーカー A が位置 30秒 に存在する
  When マーカーパネルでマーカー A のシークボタンをクリックする
  Then 再生位置が 30秒 に移動する
```

### US-2.3: マーカーの永続化

**ストーリー**: ユーザーとして、マーカーがURLごとに自動保存・復元されてほしい。再設定の手間を省くために。

**受け入れ基準**:
- [ ] マーカーは localStorage の `stream_player_loop_settings` にURLをキーとして自動保存される
- [ ] 同じURLを再度読み込むとマーカーが復元される
- [ ] URLのクエリパラメータ（t, feature, si）は正規化されて無視される
- [ ] 自動保存は2秒のデバウンスで実行される
- [ ] マーカーはプリセット間で共有される（同じ動画 = 同じマーカー）

**テストシナリオ**:

```gherkin
Scenario: マーカーが自動保存される
  Given 動画 X にマーカー A, B が設定されている
  When 2秒以上操作がない状態が続く
  Then localStorage にマーカー A, B のデータが保存される

Scenario: URL再読み込みでマーカーが復元される
  Given 動画 X のマーカーが localStorage に保存されている
  When 再度動画 X のURLを読み込む
  Then 保存されたマーカーがタイムラインとマーカーパネルに復元される

Scenario: URL正規化により同一動画が同じキーで保存される
  Given 動画 X を "https://youtube.com/watch?v=abc&t=10" で読み込み、マーカーを追加した
  When "https://youtube.com/watch?v=abc&feature=share" で再度読み込む
  Then 先ほど追加したマーカーが復元される

Scenario: デバウンスにより短時間の連続変更で保存は1回だけ
  Given 動画が読み込まれている
  When 0.5秒間隔でマーカーを3個連続追加する
  Then localStorage への書き込みは最後の追加から2秒後に1回だけ実行される
```

## Epic 3: シーケンスループ

### US-3.1: セグメントを追加する

**ストーリー**: ユーザーとして、マーカー間のセグメント（ループ区間）を定義したい。特定の区間を繰り返し練習するために。

**受け入れ基準**:
- [ ] Start/End ドロップダウン（`<select>`）からマーカーラベルを選択して「+ Add」でセグメント追加
- [ ] セグメントに任意の名前を付けられる
- [ ] Startマーカーにメモがある場合、名前欄に自動入力される
- [ ] セグメントリストに追加順に表示される

**テストシナリオ**:

```gherkin
Scenario: セグメントを追加する
  Given マーカー A（10秒）と B（30秒）が存在する
  When Start に "A"、End に "B" を選択する
    And 「+ Add」ボタンをクリックする
  Then セグメント A→B がリストに追加される

Scenario: 同一マーカー同士のセグメントは作成できない
  Given マーカー A が存在する
  When Start と End の両方に "A" を選択する
  Then 「+ Add」ボタンが disabled になる

Scenario: Startマーカーのメモがセグメント名に自動入力される
  Given マーカー A にメモ "イントロ" が設定されている
  When Start に "A" を選択する
  Then セグメント名入力欄に "イントロ" が自動入力される
```

### US-3.2: バッチ入力

**ストーリー**: ユーザーとして、複数のセグメントを一括で追加したい。効率的にシーケンスを構築するために。

**受け入れ基準**:
- [ ] 「AB,CD,EF」形式でカンマ/スペース/スラッシュ区切りのバッチ入力が可能
- [ ] 「AB:intro,CD:verse」形式でコロンの後にセグメント名を付与可能
- [ ] 大文字/小文字は区別しない
- [ ] 追加成功数と無効なペアのスキップ数がステータスに表示される
- [ ] Enter キーでもバッチ追加を実行可能

**テストシナリオ**:

```gherkin
Scenario: カンマ区切りで複数セグメントをバッチ追加する
  Given マーカー A, B, C, D が存在する
  When バッチ入力欄に "AB,CD" と入力して追加する
  Then セグメント A→B と C→D がリストに追加される
    And ステータスに "2件追加" と表示される

Scenario: コロンでセグメント名を指定する
  Given マーカー A, B が存在する
  When バッチ入力欄に "AB:intro" と入力して追加する
  Then セグメント A→B が名前 "intro" で追加される

Scenario: 存在しないマーカーラベルはスキップされる
  Given マーカー A, B が存在する（C, D は存在しない）
  When バッチ入力欄に "AB,CD" と入力して追加する
  Then セグメント A→B のみが追加される
    And ステータスに "1件追加, 1件スキップ" と表示される

Scenario: 小文字入力は大文字に変換される
  Given マーカー A, B が存在する
  When バッチ入力欄に "ab" と入力して追加する
  Then セグメント A→B が追加される
```

### US-3.3: セグメントの並び替えと削除

**ストーリー**: ユーザーとして、セグメントの再生順序を変更・削除したい。練習の流れを調整するために。

**受け入れ基準**:
- [ ] ▲/▼ ボタンでセグメントの順序を上下に移動
- [ ] × ボタンでセグメントを削除
- [ ] セグメントをクリックするとその区間の開始位置にジャンプして再生開始

**テストシナリオ**:

```gherkin
Scenario: セグメントの順序を上に移動する
  Given セグメントリストが [A→B, C→D, E→F] の順序である
  When セグメント C→D の ▲ ボタンをクリックする
  Then セグメントリストが [C→D, A→B, E→F] の順序になる

Scenario: セグメントを削除する
  Given セグメントリストに A→B と C→D がある
  When セグメント A→B の × ボタンをクリックする
  Then セグメントリストに C→D のみが残る

Scenario: セグメントをクリックしてジャンプする
  Given セグメント A（10秒）→B（30秒）が存在する
  When セグメント A→B のリスト行をクリックする
  Then 再生位置が 10秒 に移動する
    And 動画が再生開始される
```

### US-3.4: ループ再生

**ストーリー**: ユーザーとして、定義したセグメントをループ再生したい。繰り返し練習するために。

**受け入れ基準**:
- [ ] 「Start」ボタンでシーケンスループ開始
- [ ] 「Stop」ボタンでループ停止
- [ ] Ctrl+L キーでループ開始/停止をトグル
- [ ] 現在再生中のセグメントがリスト上でハイライト表示される
- [ ] タイムライン上にアクティブなセグメント区間が半透明の青色領域として表示される

**テストシナリオ**:

```gherkin
Scenario: ループを開始する
  Given セグメント A→B と C→D が定義されている
    And 動画が一時停止中である
  When 「Start」ボタンをクリックする
  Then セグメント A→B の開始位置にシークされる
    And 動画の再生が自動開始される
    And セグメント A→B がリスト上でハイライトされる

Scenario: セグメント境界で次のセグメントに遷移する
  Given Loop Sequence モードでループ再生中である
    And 現在セグメント A→B を再生中で、Bの位置が30秒
  When 再生位置が 29.85秒（= 30 - 0.15）に達する
  Then 次のセグメント C→D の開始位置にシークされる

Scenario: Ctrl+Lキーでループをトグルする
  Given セグメントが定義されている
    And ループが停止中である
  When Ctrl+L キーを押す
  Then ループが開始される
  When 再度 Ctrl+L キーを押す
  Then ループが停止される
```

### US-3.5: ループモード

**ストーリー**: ユーザーとして、ループの動作を切り替えたい。練習スタイルに合わせるために。

**受け入れ基準**:
- [ ] **Loop Sequence**: 全セグメントを順に再生し、最後まで行ったら最初に戻る
- [ ] **Loop Single**: 現在のセグメントだけを無限ループ
- [ ] **Play Once**: 全セグメントを順に1回ずつ再生し、最後に停止
- [ ] ループモードは `<select>` ドロップダウンで切替可能
- [ ] ループモードはプリセットの一部として保存される

**テストシナリオ**:

```gherkin
Scenario: Loop Sequence で末尾から先頭に戻る
  Given Loop Sequence モードで3つのセグメントが定義されている
  When 3番目のセグメントの終端に達する
  Then 1番目のセグメントの開始位置にシークされる

Scenario: Loop Single で同じセグメントを繰り返す
  Given Loop Single モードで再生中である
    And 現在セグメント A→B を再生中
  When セグメント A→B の終端に達する
  Then セグメント A→B の開始位置に再びシークされる
    And 次のセグメントには遷移しない

Scenario: Play Once で全セグメント完了後に停止する
  Given Play Once モードで2つのセグメントが定義されている
    And プレイリストが空である
  When 2番目のセグメントの終端に達する
  Then ループが停止する
    And 動画は通常再生に戻る
```

## Epic 4: オーディオエフェクト

### US-4.1: テンポ変更

**ストーリー**: ユーザーとして、再生速度を変更したい。ゆっくり練習したり、速度を上げたりするために。

**受け入れ基準**:
- [ ] テンポスライダー（`<input type="range">`）で0.25x〜2.0xの範囲で調整可能
- [ ] `[` / `]` キーで±0.05ずつ微調整
