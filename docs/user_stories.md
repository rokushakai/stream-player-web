# Stream Player ユーザーストーリー

## Epic 1: 動画の読み込みと再生

### US-1.1: URLから動画を読み込む
**ストーリー**: ユーザーとして、YouTube等のURLを入力して動画を読み込みたい。練習したい動画をすぐに開始できるように。

**受け入れ基準**:
- [ ] URL入力欄にURLを貼り付けてEnterキーまたは「Go」ボタンで読み込み開始
- [ ] 読み込み中はURL欄に「Loading...」と表示される
- [ ] 読み込み成功時、動画タイトルがURL欄下部とウィンドウタイトルに表示される
- [ ] 無効なURLや解決失敗時、エラーメッセージが赤字で表示される
- [ ] YouTube、ニコニコ動画等、yt-dlp対応サイトのURLが利用可能

### US-1.2: URL入力履歴
**ストーリー**: ユーザーとして、過去に入力したURLを履歴から選択したい。毎回URLを探す手間を省くために。

**受け入れ基準**:
- [ ] URL入力欄にドロップダウンで過去の入力履歴が表示される
- [ ] 履歴にはURLとタイトルが表示される
- [ ] 履歴はアプリ再起動後も保持される（`url_history.json`に永続化）
- [ ] 履歴から選択すると動画が読み込まれる

### US-1.3: 基本再生操作
**ストーリー**: ユーザーとして、動画の再生/一時停止、シーク、音量調整をキーボードで行いたい。マウス操作なしで素早く制御するために。

**受け入れ基準**:
- [ ] Space / K キーで再生/一時停止をトグル
- [ ] J キーで10秒戻る、L キーで10秒進む
- [ ] 左矢印で5秒戻る、右矢印で5秒進む
- [ ] Shift+左矢印で1秒戻る、Shift+右矢印で1秒進む
- [ ] 上矢印で音量+5、下矢印で音量-5
- [ ] 0〜9キーで動画の0%〜90%地点にジャンプ
- [ ] ,（カンマ）/. （ピリオド）キーで1フレーム戻る/進む（一時停止中）
- [ ] テキスト入力欄にフォーカスがある時、キーボードショートカットは無効

### US-1.4: フルスクリーン
**ストーリー**: ユーザーとして、動画をフルスクリーンで表示したい。集中して視聴するために。

**受け入れ基準**:
- [ ] F キーまたは F11 キーでフルスクリーン切替
- [ ] フルスクリーン時、URL欄・タイムライン・トランスポート・下部パネルが非表示
- [ ] Escape キーでフルスクリーン解除
- [ ] 動画エリアのダブルクリックでフルスクリーン切替
- [ ] フルスクリーン解除時、元のウィンドウサイズとレイアウトが復元される

## Epic 2: マーカー管理

### US-2.1: マーカーを追加する
**ストーリー**: ユーザーとして、動画の任意の時間位置にマーカーを追加したい。ループ区間の開始/終了地点を記録するために。

**受け入れ基準**:
- [ ] M キーまたはマーカーパネルの「+ Add Marker」ボタンで現在位置にマーカー追加
- [ ] マーカーにはアルファベット順のラベル（A, B, C, ...）が自動付与される
- [ ] マーカーには一意のUUID IDが内部的に割り当てられる
- [ ] マーカー追加後、タイムライン上にマーカー位置が表示される
- [ ] マーカーパネルのリストにマーカーが表示される（ラベル、時刻、メモ）

### US-2.2: マーカーを編集・削除する
**ストーリー**: ユーザーとして、マーカーのラベル交換やメモの編集、不要なマーカーの削除をしたい。マーカーを整理するために。

**受け入れ基準**:
- [ ] マーカーパネルで2つのマーカーのラベルを交換できる（Swap機能）
- [ ] ラベル交換してもセグメントのマーカーID参照は維持される
- [ ] マーカーにメモ（テキスト）を追加・編集できる
- [ ] メモはセグメント追加時の名前自動入力に使われる
- [ ] × ボタンでマーカーを削除できる
- [ ] 削除後、残りのマーカーのラベルは変更されない（ギャップが残る）
- [ ] マーカーをクリックするとその時間位置にシークする

### US-2.3: マーカーの永続化
**ストーリー**: ユーザーとして、マーカーがURLごとに自動保存・復元されてほしい。再設定の手間を省くために。

**受け入れ基準**:
- [ ] マーカーは`loop_settings.json`にURLをキーとして自動保存される
- [ ] 同じURLを再度読み込むとマーカーが復元される
- [ ] URLのクエリパラメータ（t, feature, si）は正規化されて無視される
- [ ] 自動保存は2秒のデバウンスで実行される
- [ ] マーカーはプリセット間で共有される（同じ動画 = 同じマーカー）

## Epic 3: シーケンスループ

### US-3.1: セグメントを追加する
**ストーリー**: ユーザーとして、マーカー間のセグメント（ループ区間）を定義したい。特定の区間を繰り返し練習するために。

**受け入れ基準**:
- [ ] Start/Endドロップダウンからマーカーラベルを選択して「+ Add」でセグメント追加
- [ ] セグメントに任意の名前を付けられる
- [ ] Start/Endのマーカーペア変更時、過去に入力した名前が自動復元される
- [ ] Startマーカーにメモがある場合、名前欄に自動入力される
- [ ] セグメントリストに追加順に表示される

### US-3.2: バッチ入力
**ストーリー**: ユーザーとして、複数のセグメントを一括で追加したい。効率的にシーケンスを構築するために。

**受け入れ基準**:
- [ ] 「AB,CD,EF」形式でカンマ/スペース/スラッシュ区切りのバッチ入力が可能
- [ ] 「AB:intro,CD:verse」形式でコロンの後にセグメント名を付与可能
- [ ] 大文字/小文字は区別しない
- [ ] 追加成功数と無効なペアのスキップ数がステータスに表示される
- [ ] Enter キーでもバッチ追加を実行可能

### US-3.3: セグメントの並び替えと削除
**ストーリー**: ユーザーとして、セグメントの再生順序を変更・削除したい。練習の流れを調整するために。

**受け入れ基準**:
- [ ] ▲/▼ ボタンでセグメントの順序を上下に移動
- [ ] × ボタンでセグメントを削除
- [ ] セグメントをクリックするとその区間の開始位置にジャンプして再生開始

### US-3.4: ループ再生
**ストーリー**: ユーザーとして、定義したセグメントをループ再生したい。繰り返し練習するために。

**受け入れ基準**:
- [ ] 「▶ Start」ボタンでシーケンスループ開始
- [ ] 「■ Stop」ボタンでループ停止
- [ ] Ctrl+L キーでループ開始/停止をトグル
- [ ] 現在再生中のセグメントがハイライト表示される
- [ ] タイムライン上にアクティブなセグメント区間が視覚的に表示される

### US-3.5: ループモード
**ストーリー**: ユーザーとして、ループの動作を切り替えたい。練習スタイルに合わせるために。

**受け入れ基準**:
- [ ] **Loop Sequence**: 全セグメントを順に再生し、最後まで行ったら最初に戻る
- [ ] **Loop Single**: 現在のセグメントだけを無限ループ
- [ ] **Play Once**: 全セグメントを順に1回ずつ再生し、最後に停止
- [ ] ループモードはドロップダウンで切替可能
- [ ] ループモードはプリセットの一部として保存される

## Epic 4: オーディオエフェクト

### US-4.1: テンポ変更
**ストーリー**: ユーザーとして、再生速度を変更したい。ゆっくり練習したり、速度を上げたりするために。

**受け入れ基準**:
- [ ] テンポスライダーで0.25x〜2.0xの範囲で調整可能
- [ ] `[` / `]` キーで±0.05ずつ微調整
- [ ] `<` / `>` キー（Shift+,/.）で±0.25ずつ大幅調整
- [ ] プリセットボタン（0.5x, 0.75x, 1.0x, 1.25x, 1.5x）でワンクリック設定
- [ ] テンポ変更はピッチに影響しない（タイムストレッチ）

### US-4.2: トランスポーズ（移調）
**ストーリー**: ユーザーとして、再生ピッチを半音単位で変更したい。楽器のキーに合わせるために。

**受け入れ基準**:
- [ ] トランスポーズスライダーで-12〜+12半音の範囲で調整可能
- [ ] `-` / `=` キーで±1半音ずつ調整
- [ ] ピッチ変更は再生速度に影響しない
- [ ] 「Reset」ボタンでテンポ・トランスポーズを初期値に戻せる

---

## Epic 5: プリセット（プログラム）

### US-5.1: プリセットを保存する
**ストーリー**: ユーザーとして、現在のセグメント構成とループモードを名前付きプリセットとして保存したい。同じ動画で異なる練習パターンを使い分けるために。

**受け入れ基準**:
- [ ] Sequenceパネルのプリセット名入力欄に名前を入力し「Save As」で保存
- [ ] プリセットにはセグメント構成とループモードが含まれる
- [ ] マーカーはプリセット間で共有される（プリセットには含まれない）
- [ ] 「default」プリセットは常に存在する
- [ ] プリセットはURLごとに`loop_settings.json`に永続化される
- [ ] 旧フォーマット（プリセット未対応）のデータは自動的にマイグレーションされる

### US-5.2: プリセットを切り替える
**ストーリー**: ユーザーとして、保存済みプリセットをドロップダウンから選択して切り替えたい。練習パターンを素早く変更するために。

**受け入れ基準**:
- [ ] Sequenceパネルのプリセットドロップダウンに利用可能なプリセット一覧が表示される
- [ ] プリセットを選択すると、そのセグメント構成とループモードが読み込まれる
- [ ] 切替前のプリセットは自動保存される
- [ ] マーカーはプリセット切替時も維持される

### US-5.3: プリセットを削除する
**ストーリー**: ユーザーとして、不要なプリセットを削除したい。プリセットリストを整理するために。

**受け入れ基準**:
- [ ] 「Del」ボタンで現在選択中のプリセットを削除
- [ ] 「default」プリセットは削除不可
- [ ] 削除後は自動的に「default」プリセットに切り替わる

---

## Epic 6: プレイリスト

### US-6.1: プレイリストに追加する
**ストーリー**: ユーザーとして、現在の動画をプレイリストに追加したい。複数動画の連続再生を準備するために。

**受け入れ基準**:
- [ ] 「+ Add Current」ボタンで現在の動画とプリセットをプレイリストに追加
- [ ] プレイリストには動画タイトル、URL、プリセット名が記録される
- [ ] プレイリストはアプリ再起動後も保持される（`playlist.json`に永続化）
- [ ] default以外のプリセットが設定されている場合、プリセット名が黄色で表示される

### US-6.2: プレイリストの操作
**ストーリー**: ユーザーとして、プレイリストの順序変更、項目削除、全消去をしたい。再生順を調整するために。

**受け入れ基準**:
- [ ] ▲/▼ ボタンで項目の順序を上下に移動
- [ ] × ボタンで項目を削除
- [ ] 「Clear」ボタンで全項目を削除
- [ ] 項目をクリックするとその動画が読み込まれ、対応するプリセットが適用される
- [ ] 現在再生中の項目は青いハイライトと▶インジケーターで表示される

### US-6.3: Play Once自動遷移
**ストーリー**: ユーザーとして、Play Onceモードで全セグメント完了後に自動的に次の動画に遷移してほしい。ハンズフリーで連続練習するために。

**受け入れ基準**:
- [ ] ループモードが「Play Once」で全セグメント再生完了時、`play_once_finished`イベントが発火
- [ ] プレイリストに次の項目がある場合、自動的に次の動画URLが読み込まれる
- [ ] 次の動画のプリセット設定（セグメント + ループモード）が自動復元される
- [ ] セグメントが存在する場合、ループが自動開始される
- [ ] 一時停止中だった場合、自動的に再生が開始される
- [ ] プレイリスト最後の動画のPlay Once完了時は停止する（無限ループしない）
- [ ] 動画切替中のエラー（ネットワーク障害等）は適切にハンドリングされる

## Epic 7: タイムラインとビジュアル

### US-7.1: タイムライン表示
**ストーリー**: ユーザーとして、動画の進行状況とマーカー・セグメント位置をタイムラインで視覚的に確認したい。現在位置を把握するために。

**受け入れ基準**:
- [ ] タイムラインキャンバスに再生位置のプログレスバーが表示される
- [ ] マーカー位置がタイムライン上に表示される
- [ ] アクティブなセグメント区間がハイライト表示される
- [ ] タイムラインクリックで任意の位置にシーク可能

### US-7.2: トランスポートバー
**ストーリー**: ユーザーとして、再生時間と操作ボタンをトランスポートバーで確認・操作したい。

**受け入れ基準**:
- [ ] 現在時間 / 総時間が表示される
- [ ] 再生/一時停止ボタンが利用可能
- [ ] フルスクリーンボタン（⛶）が利用可能

## Epic 8: データ永続化

### US-8.1: 設定の自動保存
**ストーリー**: ユーザーとして、マーカーやセグメント設定が自動保存されてほしい。手動保存を忘れてデータを失わないために。

**受け入れ基準**:
- [ ] マーカー変更時、2秒のデバウンス後に自動保存
- [ ] セグメント変更時、2秒のデバウンス後に自動保存
- [ ] 動画切替時に即座に保存
- [ ] アプリ終了時に即座に保存
- [ ] 設定復元中（`_restoring`フラグ）は自動保存が抑制される

### US-8.2: データフォーマットの後方互換性
**ストーリー**: ユーザーとして、アプリのアップデート後も既存の設定データが失われないでほしい。

**受け入れ基準**:
- [ ] 旧フォーマット（プリセットなし）の`loop_settings.json`を読み込み時に自動マイグレーション
- [ ] マイグレーション後、既存データは「default」プリセットに格納される
- [ ] マイグレーションは非破壊的（データ損失なし）
- [ ] `playlist.json`のpresetフィールドが未設定の場合は「default」として扱う

---

## Epic 9: ウィンドウとレイアウト

### US-9.1: リサイズ可能なレイアウト
**ストーリー**: ユーザーとして、動画エリアと下部パネルの比率をドラッグで調整したい。画面サイズに合わせたレイアウトにするために。

**受け入れ基準**:
- [ ] PanedWindowのサッシュをドラッグして動画エリアと下部パネルの比率を変更可能
- [ ] 動画エリアの最小高さ150px、下部パネルの最小高さ100px
- [ ] 下部パネルは3カラム構成（マーカー、シーケンス、プレイリスト）
- [ ] 最小ウィンドウサイズ: 900x650

### US-9.2: ダークテーマ
**ストーリー**: ユーザーとして、ダークテーマで表示されてほしい。長時間の使用で目が疲れないために。

**受け入れ基準**:
- [ ] アプリ起動時にダークモード（customtkinter）が適用される
- [ ] カラーテーマは「blue」

## 非機能要件

### NFR-1: パフォーマンス
- [ ] 動画読み込みはバックグラウンドスレッドで実行され、UIがフリーズしない
- [ ] セグメント境界検知は0.15秒の閾値（SEEK_THRESHOLD）で動作する
- [ ] 自動保存のデバウンスにより、頻繁なディスクI/Oを防止する

### NFR-2: スレッド安全性
- [ ] GUIの更新は`widget.after(0, callback)`でメインスレッドにディスパッチされる
- [ ] SequenceLooperの状態管理はRLockで保護される

### NFR-3: 堅牢性
- [ ] ファイルの読み書きエラーはキャッチされ、アプリがクラッシュしない
- [ ] 不正なURL入力時はエラーメッセージが表示される
